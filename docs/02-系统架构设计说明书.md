# 《02-系统架构设计说明书.md》

## 1. 总体架构
采用“网关 + 业务服务 + 渠道适配 + 中间件”分层架构。

- API Gateway：统一鉴权、限流、路由
- Payment Service：支付核心业务
- Tenant Service：租户与配置管理
- Channel Adapter：微信/支付宝/Stripe 等适配
- Notify Service：异步通知处理
- Reconciliation Service：对账服务
- Admin Portal：后台管理系统

## 2. 技术栈
- .NET 8 / ASP.NET Core
- EF Core 8
- PostgreSQL（或 SQL Server）
- Redis
- RabbitMQ（或 Kafka）
- OpenTelemetry + Prometheus + Grafana + ELK

## 3. 部署架构
- Kubernetes 多副本部署（无状态 API）
- Ingress/Nginx 负载均衡
- HPA 自动扩缩容
- Redis 高可用（哨兵或集群）
- DB 主从架构 + 定期备份

## 4. 多租户隔离策略
- 表级隔离：所有业务表强制 TenantId
- 应用层隔离：EF Core 全局查询过滤器
- 配置隔离：租户渠道参数独立加密存储
- 资源隔离：按租户限流阈值与配额

## 5. 高可用设计
- 服务无状态，多实例横向扩展
- 第三方调用使用熔断/重试/超时
- 消息队列削峰填谷
- 回调失败进入重试队列 + 死信队列

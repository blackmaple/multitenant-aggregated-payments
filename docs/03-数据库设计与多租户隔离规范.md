# 《03-数据库设计与多租户隔离规范.md》

## 1. 表设计原则
- 每个核心业务表必须包含 TenantId
- 所有金额使用 `decimal(18,2)`（或更高精度）
- 使用逻辑状态字段，不做硬删除
- 审计字段：CreatedAt/UpdatedAt/CreatedBy/UpdatedBy

## 2. 核心表结构（简化）

## 2.1 tenants
- Id (bigint, PK)
- TenantCode (varchar 64, unique)
- TenantName (varchar 128)
- Status (smallint)
- CreatedAt (timestamp)

索引：
- UK_Tenants_TenantCode

## 2.2 tenant_apps
- Id (bigint, PK)
- TenantId (bigint, indexed)
- AppId (varchar 64)
- AppName (varchar 128)
- NotifyUrl (varchar 512)
- Status (smallint)

索引：
- IDX_TenantApps_TenantId
- UK_TenantApps_TenantId_AppId

## 2.3 channel_configs
- Id (bigint, PK)
- TenantId (bigint, indexed)
- AppId (varchar 64)
- ChannelCode (varchar 32)
- ConfigCipherText (text)  // 加密配置
- Version (int)
- Status (smallint)

索引：
- UK_ChannelConfigs_TenantId_AppId_ChannelCode

## 2.4 payment_orders
- Id (bigint, PK)
- TenantId (bigint, indexed)
- MerchantOrderNo (varchar 64)
- PlatformOrderNo (varchar 64, unique)
- ChannelCode (varchar 32)
- Amount (decimal 18,2)
- Currency (varchar 8)
- Status (varchar 32)
- IdempotencyKey (varchar 64)
- CreatedAt (timestamp)
- ExpireAt (timestamp)

索引：
- UK_PaymentOrders_TenantId_MerchantOrderNo
- IDX_PaymentOrders_TenantId_Status_CreatedAt
- IDX_PaymentOrders_PlatformOrderNo

## 2.5 payment_transactions
- Id (bigint, PK)
- TenantId (bigint)
- PlatformOrderNo (varchar 64, indexed)
- ChannelOrderNo (varchar 128)
- RequestRaw (text)
- ResponseRaw (text)
- Status (varchar 32)
- CreatedAt (timestamp)

## 2.6 refund_orders
- Id (bigint, PK)
- TenantId (bigint)
- RefundOrderNo (varchar 64, unique)
- PlatformOrderNo (varchar 64, indexed)
- Amount (decimal 18,2)
- Status (varchar 32)
- CreatedAt (timestamp)

## 2.7 webhook_events
- Id (bigint, PK)
- TenantId (bigint)
- ChannelCode (varchar 32)
- EventId (varchar 128)
- PayloadRaw (text)
- VerifyStatus (smallint)
- ProcessStatus (smallint)
- RetryCount (int)
- CreatedAt (timestamp)

索引：
- IDX_WebhookEvents_TenantId_ChannelCode_CreatedAt
- UK_WebhookEvents_ChannelCode_EventId

## 3. 多租户数据访问规范
- Repository 层必须传递 TenantId
- 管理员跨租户查询需显式接口 + 审计日志
- SQL 禁止无 TenantId 条件查询核心交易表

# 《04-缓存与高可用设计规范.md》

## 1. 缓存使用目标
- 降低数据库读压力
- 减少配置读取时延
- 防止重复请求造成重复交易

## 2. 缓存模型
### 2.1 Cache Aside
1. 先读 Redis  
2. 未命中读 DB  
3. 回填 Redis 并设置 TTL  

### 2.2 二级缓存
- L1：MemoryCache（进程内，毫秒级）
- L2：Redis（分布式共享）

## 3. 缓存 Key 规范
- `tenant:{tenantId}:app:{appId}:channel:{channel}:config`
- `pay:idempotent:{tenantId}:{merchantOrderNo}:{idemKey}`
- `route:tenant:{tenantId}:channel:policy`
- `notify:dedup:{channel}:{eventId}`

## 4. TTL 策略
- 配置缓存：30 min + 随机 0~300s
- 幂等缓存：24 h
- 回调去重：48 h
- 路由策略：10 min

## 5. 缓存异常防护
- 缓存穿透：空值缓存 + 参数校验
- 缓存击穿：热点 key 分布式锁
- 缓存雪崩：随机过期 + 分批预热

## 6. 高可用策略
- API 多副本 + LB
- Redis 集群 + 持久化
- DB 主从 + 读写分离（按需）
- MQ 重试队列 + 死信队列
- 第三方渠道调用熔断降级

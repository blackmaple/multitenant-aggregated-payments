# 《01-产品需求说明书-多租户聚合支付.md》

## 1. 文档信息
- 文档版本：v1.0
- 编写日期：2026-06-30
- 适用系统：多租户聚合支付平台（.NET 8）
- 读者对象：产品经理、架构师、后端工程师、测试工程师、运维工程师

## 2. 项目背景
为满足 SaaS 商户平台中“多商户、多支付渠道、统一管理”的需求，建设一个多租户聚合支付平台，支持租户快速接入支付能力，降低渠道对接复杂度，提升稳定性与可观测性。

## 3. 项目目标
1. 提供统一支付能力：下单、退款、查询、回调、对账  
2. 实现租户隔离：数据隔离、配置隔离、限流隔离  
3. 支持高并发高可用：负载均衡、缓存加速、故障隔离  
4. 提供标准 SDK：快速接入第三方支付渠道  
5. 建设审计与监控体系：交易可追踪、异常可告警

## 4. 范围定义
### 4.1 In Scope
- 租户管理（开户、禁用、配额）
- 应用管理（AppId、签名、公私钥）
- 支付单管理（创建、关闭、查询）
- 退款管理（申请、查询）
- 回调通知（验签、幂等处理）
- 对账任务（日账单拉取、差异记录）
- 平台管理后台（配置、报表、告警）

### 4.2 Out of Scope
- 清结算系统（资金清算）
- 信贷分期业务
- 国际复杂税务处理

## 5. 角色与权限
- 平台管理员：管理租户、渠道模板、全局策略
- 租户管理员：管理本租户应用和支付配置
- 业务系统：调用支付 API、查询订单
- 运营人员：查看报表、下载对账单

## 6. 核心业务流程
1. 租户入驻并配置支付渠道参数  
2. 业务系统调用统一下单接口  
3. 系统根据租户路由策略选择支付渠道  
4. 用户完成支付后，渠道异步回调平台  
5. 平台验签并更新订单状态  
6. 平台通知业务系统支付结果  
7. 每日执行对账任务并生成差异报告

## 7. 非功能需求
- 可用性：99.9%+
- 下单接口性能：P95 < 150ms（不含渠道侧耗时）
- 安全要求：TLS1.2+、敏感字段加密、日志脱敏
- 合规要求：全链路审计、操作留痕
- 扩展性：新增渠道不影响核心流程

## 8. 验收标准（示例）
- 支持至少 3 个渠道并完成统一下单
- 幂等重复下单不产生重复订单
- 回调重复通知不重复更新业务状态
- Redis 缓存命中率 > 85%
- 单渠道故障不影响其他渠道支付
